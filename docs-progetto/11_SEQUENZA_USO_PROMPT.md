# Sequenza uso prompt

## Prima di lanciare Codex
1. Metti questi file nella root del progetto
2. Apri e rileggi i documenti 01–07
3. Usa 08 come prompt master
4. Crea ogni task con il template 09
5. Segui l’ordine del file 10

## Strategia consigliata
- prompt non troppo piccoli
- prompt non troppo larghi
- 1 modulo consistente per volta
- controlla il diff dopo ogni esecuzione

## Regola di qualità
Se un task Codex rompe qualcosa di già buono:
- non proseguire
- correggi
- poi lancia il task successivo
